# Instructions
Perform a full production-grade audit of this application without making assumptions. Review the entire codebase, database schema, environment configuration, authentication logic, API routes, frontend/backend boundaries, dependencies, error handling, logging, deployment configuration, and any integrations.

Your goal is to identify anything that could create security risk, instability, poor scalability, bad maintainability, data loss, privacy exposure, or production-readiness issues.

Please complete the following:

1. Application Security Review

- Review authentication, authorization, session handling, password handling, role-based access, protected routes, API access control, input validation, output sanitization, CORS, CSRF, rate limiting, file upload handling, secrets management, and error exposure.

- Identify any insecure defaults, exposed credentials, overly permissive access, missing validation, or weak permission boundaries.

- Check whether admin-only functionality is properly protected on both frontend and backend.

2. Database-Level Security & Optimization

- Review database schema, relationships, indexes, constraints, foreign keys, cascading behavior, nullable fields, default values, migrations, and query patterns.

- Identify risks of SQL injection, unsafe raw queries, inefficient queries, missing indexes, N+1 issues, data duplication, weak constraints, and potential data integrity problems.

- Recommend improvements for production-grade reliability, performance, and data protection.

- Confirm that sensitive data is not stored unnecessarily, and if it is, recommend encryption, hashing, masking, or access restrictions.

3. Code Quality & Refactoring

- Review the codebase for duplication, overly complex logic, inconsistent patterns, dead code, unused files, naming issues, poor separation of concerns, and hardcoded values.

- Recommend refactors that improve maintainability without changing existing functionality.

- Prioritize changes that reduce technical debt and improve long-term scalability.

4. Production Readiness

- Review deployment configuration, environment variables, build scripts, error handling, logging, monitoring readiness, health checks, fallback states, loading states, and failure handling.

- Identify anything that may work in development but fail in production.

- Check whether the application has proper handling for failed API calls, failed database connections, expired sessions, invalid user input, and unexpected server errors.

5. Dependency & Package Audit

- Review all dependencies for outdated, unnecessary, risky, or vulnerable packages.

- Recommend package updates or removals where appropriate.

- Do not introduce unnecessary libraries unless there is a clear production-grade reason.

6. Privacy & Data Handling

- Review how user, customer, candidate, company, email, phone, document, or other sensitive data is collected, stored, transmitted, displayed, and deleted.

- Identify any places where private data may be leaked through logs, errors, frontend state, URLs, browser storage, or API responses.

- Recommend improvements aligned with secure business application practices.

7. Performance & Scalability

- Review frontend performance, backend response patterns, database calls, caching opportunities, asset loading, pagination, filtering, and expensive operations.

- Identify anything that could slow down as usage grows.

- Recommend practical optimizations suitable for this application’s current stage.

8. Testing & Validation

- Review existing tests or lack of tests.

- Recommend critical tests for authentication, permissions, database operations, core workflows, form validation, error states, and API endpoints.

- If appropriate, add or suggest lightweight tests that protect the most important workflows.

9. Documentation & Maintainability

- Review whether setup, deployment, environment variables, database migrations, and major workflows are clearly documented.

- Recommend documentation improvements that would help future developers or AI agents safely work on this project.

10. Final Output Required

Provide a structured audit report with the following sections:

A. Executive Summary

B. Critical Issues

C. High-Priority Issues

D. Medium-Priority Issues

E. Low-Priority Improvements

F. Database Findings

G. Security Findings

H. Refactoring Opportunities

I. Production Readiness Checklist

J. Recommended Implementation Plan

For each finding, include:

- Issue

- Why it matters

- Risk level

- Exact file or area affected

- Recommended fix

- Whether you can safely fix it now or whether it requires approval

Important instructions:

- Do not make destructive changes.

- Do not delete data.

- Do not modify database schema without explaining the migration impact first.

- Do not change authentication, authorization, or production configuration without first explaining the risk and expected result.

- Preserve all existing business logic unless a change is clearly needed for security, reliability, or maintainability.

- Prioritize safe, incremental improvements.

- Before applying changes, show me the audit findings and recommended action plan.

- After I approve, implement fixes in small batches and summarize exactly what changed.
